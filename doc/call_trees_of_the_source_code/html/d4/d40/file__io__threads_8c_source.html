40 // In order to overlap disk I/O with computation, we have defined
41 // several routines which can do disk I/O in a non-blocking way. The
42 // first launches a thread (using pthreads), which writes to
43 // disk using fwrite. The second version, attempts to use the linux
44 // system call aio_write, which produces a similar result, but is not
45 // able to use buffered I/O resulting in slower performance. Both
46 // these routines must copy the memory region they are saving, because
47 // it will change as the output is uptdated during the next time
48 // step. This could be solved by rotating pointers, but this is not
49 // very easy in Fortran (especially without changing the fortran code)

187  if (rc != 0 ) exit_error("Adjoint thread: setting thread state failed");
188
