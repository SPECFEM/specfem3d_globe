28 ! XCLIP_SEM
29 !
30 ! USAGE
31 ! mpirun -np NPROC bin/xclip_sem MIN_VAL MAX_VAL KERNEL_NAMES INPUT_DIR OUTPUT_DIR
32 !
33 !
34 ! COMMAND LINE ARGUMENTS
35 ! MIN_VAL - threshold below which array values are clipped
36 ! MAX_VAL - threshold above which array values are clipped
37 ! KERNEL_NAMES - one or more material parameter names separated by commas
38 ! INPUT_DIR - directory from which arrays are read
39 ! OUTPUT_DIR - directory to which clipped array are written
40 !
41 !
42 ! DESCRIPTION
43 ! For each name in KERNEL_NAMES, reads kernels from INPUT_DIR, applies
44 ! thresholds, and writes the resulting clipped kernels to OUTPUT_DIR.
45 !
46 ! KERNEL_NAMES is comma-delimited list of material names,
47 ! e.g. 'alphav_kernel,alphah_kernel'
48 !
49 ! Files written to OUTPUT_DIR have the suffix 'clip' appended,
50 ! e.g. proc***alphav_kernel.bin becomes proc***alphav_kernel_clip.bin
51 !
52 ! This program's primary use case is to clip kernels. It can be used though on
53 ! any "reg1" array, i.e. any array of dimension
54 ! (NGLLX,NGLLY,NGLLZ,NSPEC_CRUST_MANTLE). The region suffix must be included
55 ! explicitly in all names supplied through the KERNEL_NAMES arugment.
56 !
57 ! This is a parrallel program -- it must be invoked with mpirun or other
58 ! appropriate utility. Operations are performed in embarassingly-parallel
59 ! fashion.
